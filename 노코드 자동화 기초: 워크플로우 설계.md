# 프로젝트 1: Make vs Zapier 자동화 도구 비교

## 1. 프로젝트 개요

### 목표
동일한 자동화 시나리오를 **Make**와 **Zapier**로 각각 구현하고,
두 노코드 자동화 도구의 구조·기능·사용성 차이를 비교 분석한다.

### 시나리오: 설문 응답 자동 분류 및 안내 메일 발송
1. Google Form으로 점수가 포함된 응답을 수집한다.
2. 점수 기준(80점)으로 **합격 / 불합격**을 자동 분기한다.
3. 결과에 따라 Google Sheets의 해당 탭(합격/불합격)에 행을 추가한다.
4. 응답자 이메일로 합격/불합격 안내 메일을 자동 발송한다.

### 사용 도구
| 구분 | 내용 |
|---|---|
| 자동화 도구 | Make (무료 플랜), Zapier (무료 플랜) |
| 연동 서비스 | Google Forms, Google Sheets, Gmail |

## 3. 비교 분석

### 3-1. 주요 차이점

| 비교 항목 | Make | Zapier |
|---|---|---|
| **분기 처리** | Router 모듈로 시나리오 1개 안에서 분기 | 무료 플랜은 분기 불가 → Zap 2개로 분리 |
| **필터 연산자** | `≥ (이상)` 등 다양한 연산자 지원 | `≥` 없음 → `Greater(less) than 79.9`로 우회 |
| **즉시 테스트** | "Run once"로 바로 실행 가능 | 실행 과정이 까다로움 |
| **테스트 데이터** | 실행 시 자동으로 다음으로 넘어감 | 샘플 레코드를 수동 선택해야 함 |
| **UI 형태** | 원형 모듈을 선으로 잇는 시각적 캔버스 | 위에서 아래로 흐르는 단계형 리스트 |

### 3-2. 각 도구의 장단점

**Make**
- 👍 장점: 시각적 캔버스로 전체 흐름 파악이 쉬움, Router로 복잡한 분기를 하나의 시나리오에서 처리, 즉시 실행 테스트 편리
- 👎 단점: 처음에는 모듈/매핑 개념 학습이 필요, 초보자에게 진입장벽이 상대적으로 높음

**Zapier**
- 👍 장점: 단계형 UI로 직관적이고 배우기 쉬움, 연동 가능한 앱 수가 많음, Copilot 등 AI 보조 기능
- 👎 단점: 무료 플랜 제약이 큼(분기 불가 등), 같은 시나리오도 Zap을 여러 개 만들어야 해 관리 포인트 증가

---

## 4. 결론

동일한 자동화를 두 도구로 구현한 결과, **기능적으로는 모두 목표를 달성**할 수 있었다.
다만 접근 방식에서 뚜렷한 차이가 있었다.

- **Make**: 하나의 시나리오에서 분기까지 처리하는 통합적 설계가 가능하고, 테스트가 빠르다.
  → 조건 분기가 있는 **복잡한 워크플로우**에 유리
- **Zapier**: 단계형 UI로 진입장벽이 낮지만, 무료 플랜의 제약(분기 불가)으로
  같은 작업에 Zap 2개가 필요했다.
  → **단순하고 직선적인 자동화**를 빠르게 만들 때 유리
